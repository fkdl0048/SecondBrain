## 작업 범위: 5단계 Monetization 모듈 (이번 세션은 Monetization만 다룬다)
프로젝트 루트의 CLAUDE.md 규칙을 반드시 따를 것. 특히:
- Monetization은 Core만 의존(Data 의존 금지). Monetization.asmdef가 Core.asmdef만 참조하게 설정.
- 게임 고유 코드는 넣지 말 것. 인터페이스 + 더미 구현 + 퍼사드만.
- 실제 SDK(AdMob / Unity Ads / Unity IAP / Google UMP)는 설치하지 말 것.
  인터페이스로만 정의하고, 실제 연결은 게임별로 한다.

### 0) 설계 결정 (Plan에서 먼저 확정)
- Monetization은 Data를 의존할 수 없으므로 '광고 제거' 상태/동의 상태를 직접 저장하지 않는다.
- 권장안: Monetization은 인터페이스 + 런타임 상태 + 이벤트만 책임지고,
  실제 영속 저장은 Game(조립) 레이어가 Data와 연결한다.
  (대안: Core만 의존하는 IMonetizationStorage 인터페이스를 두고 인메모리 더미를 제공,
   Data 기반 구현은 Game에서 주입)
  어느 쪽이 나은지 Plan에서 의견 줘.

Assets/_Project/Scripts/Monetization 아래에 Monetization.asmdef와 함께 다음을 만들어줘.

### 1) 광고 추상화 (IAdService)
- 전면(Interstitial), 보상형(Rewarded), 배너(Banner) 지원
- 로드/표시는 비동기(콜백 또는 async), 결과를 enum/이벤트로 통지
  (Loaded / Failed / Shown / Closed / RewardEarned)
- 보상형은 보상 지급 여부 콜백 제공
- 배너는 표시 / 숨김 / 위치(상단·하단) 지정

### 2) IAP 추상화 (IPurchaseService)
- 초기화 + 상품 카탈로그(상품 ID 목록) 등록
- 상품 조회(현지화된 가격 문자열 포함), 구매, 복원(Restore — iOS 필수)
- 구매 결과 콜백/이벤트(Success / Failed / Pending / Restored)
- 영수증 검증 훅(인터페이스만 — 실제 검증은 게임/서버 몫)

### 3) 동의 추상화 (IConsentService)
- 동의 요청 / 상태 조회(Required / Obtained / NotRequired)
- 광고 초기화는 동의가 해소된 뒤 진행되도록 게이팅
- (Google UMP를 염두에 두되, 실제 SDK는 연결하지 않음)

### 4) 더미 구현 (SDK 없이 동작)
- 위 세 인터페이스의 더미 구현 제공
- 전면: 로그 후 즉시 닫힘 처리 / 보상형: 짧은 지연 후 보상 지급 성공 / 배너: 표시 상태만 토글
- IAP: 구매 즉시 성공, 가짜 현지화 가격 문자열 반환
- 동의: 즉시 '동의 완료' 반환
- 네이밍: 인터페이스는 IXxx, 더미는 DummyXxx

### 5) MonetizationManager (퍼사드, 영속 싱글톤)
- Game 레이어가 호출하는 단일 진입점. 내부에 현재 IAdService / IPurchaseService / IConsentService 보유
- Game은 SDK를 직접 호출하지 않고 이 퍼사드만 사용
- '광고 제거' 런타임 플래그 관리 + 변경 시 Core 이벤트 버스로 발행
- 광고 제거 상태면 전면/배너 자동 비표시(보상형은 유지)

### 6) 프로바이더 교체 구조
- 더미를 기본으로 하고, 실제 구현을 주입/교체할 수 있는 구조
  (초기화 시 주입 또는 설정 ScriptableObject로 선택)
- 게임별로 실제 AdMob / Unity IAP / UMP를 끼우는 절차를 README에 문서화

### 공통 요구사항
- 네임스페이스: Bedrock.Monetization
- public API는 XML 주석으로 문서화
- Monetization 폴더에 README.md로
  광고 표시/보상형/구매/복원/동의 흐름, 광고 제거 처리, 실제 SDK 연결 절차 정리

### 검증
- 더미 상태에서 전면·보상형·배너, 구매·복원, 동의 흐름이 콜백/이벤트로 동작하는지
- '광고 제거' 토글 시 전면·배너가 막히고 보상형은 유지되는지
- 예시 호출 코드가 SDK가 아니라 퍼사드/인터페이스만 통하는지
