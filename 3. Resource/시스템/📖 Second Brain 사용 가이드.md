# 📖 Second Brain 사용 가이드

> 이 문서는 이 볼트가 어떻게 작동하는지 설명해요.  
> 막힐 때마다 여기서 시작하세요.

---

## 전체 구조 한눈에 보기

```
SecondBrain/
├── 0. Inbox/       ← 빠른 캡처 공간 (처리 전 임시 보관)
├── Daily/          ← 날짜별 일일 노트
├── 1. Project/     ← 목표와 기한이 있는 일
├── 2. Area/        ← 지속적으로 관리하는 영역
├── 3. Resource/    ← 나중에 쓸 참고 자료
├── 4. Archive/     ← 완료되거나 더 이상 안 쓰는 것
└── Templates/      ← 노트 템플릿 모음
```

---

## 각 폴더의 역할

### 📥 0. Inbox — "일단 던져두는 곳"

Inbox는 **처리 전 임시 보관함**이에요. 생각이 날 때, 링크를 저장할 때, 뭔가 적어두고 싶을 때 고민 없이 여기에 던져두세요.

**핵심 규칙**: Inbox는 자주 비워야 해요. **주 1회** Inbox를 열고 각 노트를 알맞은 폴더로 옮기거나 삭제하세요. Inbox가 쌓이기 시작하면 시스템이 무너집니다.

> ✅ 해도 되는 것: 빠른 메모, 나중에 볼 링크, 아직 분류 안 된 생각  
> ❌ 하면 안 되는 것: Inbox를 영구 보관함으로 쓰기

---

### 📅 Daily/ — "오늘의 기록"

날짜별 일일 노트 전용 폴더예요. `Daily Note` 템플릿을 사용하세요.

일일 노트는 **할 일 + 오늘의 메모 + 짧은 회고**로 구성돼요. 완벽하게 쓰려 하지 말고, 당일의 흔적을 남긴다는 느낌으로 쓰세요.

---

### 🚀 1. Project/ — "지금 하고 있는 일"

**기한이 있고 목표가 명확한 것**들이 여기에 들어와요.

```
1. Project/
├── GameOver/
│   ├── 📋 GameOver Hub.md   ← 시작점
│   ├── 개발일지/
│   ├── 기획/
│   ├── 기술메모/
│   └── 회의록/
└── MaskOfTime/
    └── 📋 MaskOfTime Hub.md
```

각 프로젝트는 **Hub 노트**를 중심으로 운영해요. Hub 노트에는 현재 상태, 폴더 구조, 빠른 링크가 담겨 있어요. 프로젝트를 열 때는 항상 Hub 노트에서 시작하세요.

**Project vs Area 차이:**  
- Project: "GameOver를 이번 달까지 알파 버전 완성하기" → **기한 O**  
- Area: "게임 개발 실력을 계속 키우기" → **기한 X**

---

### 🌱 2. Area/ — "지속적으로 관리하는 영역"

기한은 없지만 **꾸준히 신경 써야 하는 것**들이에요.

```
2. Area/
├── 개인글/
│   ├── 독후감/    ← 읽은 책 정리
│   ├── 아이디어/  ← 게임 아이디어, 창작 아이디어
│   └── 에세이/    ← 개인적인 글
└── 북클럽/
```

Area는 "완성"이 없어요. 꾸준히 쌓아가는 공간이에요.  
독후감, 아이디어 노트, 에세이는 각각 템플릿을 사용하면 쓰기 훨씬 편해요.

---

### 📚 3. Resource/ — "나중에 쓸 재료"

지금 당장 활용하진 않지만, **언젠가 참고할 것들**을 모아두는 곳이에요.

```
3. Resource/
├── GameDev/        ← NDC 영상 정리, PM 자료
│   └── PM/
├── GameLogic/      ← NUnit, VContainer 등 기술 자료
├── GameReferences/ ← 게임 레퍼런스, 아트 스타일
└── 시스템/         ← 이 볼트 사용 방법 (지금 이 파일)
```

**Resource vs Area 차이:**  
- Resource: "NDC 영상 링크 모음" → 내가 수집한 외부 자료  
- Area: "게임 개발 실력 향상" → 내가 관리하는 내 영역

---

### 🗄 4. Archive/ — "더 이상 안 쓰는 것"

완료된 프로젝트, 지나간 계획, 쓰임이 끝난 노트들이 여기에 와요.  
삭제하기 아깝지만 방해가 되는 것들을 여기에 보내면 돼요.

> Archive는 휴지통이 아니에요. 완료된 것들의 기록이에요. 나중에 다시 꺼낼 수 있어요.

---

## Zettelkasten이란?

> "메모들이 서로 연결되어 새로운 생각을 만들어내는 방식"

독일의 사회학자 니클라스 루만이 90,000개의 메모로 70권의 책을 쓴 방법이에요.

### 핵심 원칙 3가지

**1. Atomic (원자적)** — 노트 하나에 생각 하나만  
여러 생각을 하나의 노트에 욱여넣지 마세요. "유니티 이벤트 시스템"에 대한 노트와 "VContainer 사용법" 노트는 분리돼야 해요.

**2. Own Words (자기 말로)** — 복붙 금지, 이해한 것을 자기 언어로  
NDC 영상을 보고 링크만 저장하면 기억에 안 남아요. "이 발표에서 내가 배운 것은..."으로 시작하는 메모가 진짜 메모예요.

**3. Link (연결)** — 다른 노트와 연결하기  
노트를 쓸 때 `[[다른 노트]]`로 연결하세요. 연결이 쌓이면 Graph View에서 나만의 지식 지도가 보여요.

### 실제로 어떻게 쓰냐면

1. NDC 영상을 보다가 "PM은 게임이 완성되게 하는 사람"이라는 말이 인상 깊었다
2. `ZettelKasten` 템플릿으로 Inbox에 메모 하나 만든다
3. 나중에 Inbox 처리 시, 이미 있는 `[[전지적 참견시점 게임 개발 PM 정리]]` 노트와 연결한다
4. 비슷한 생각이 쌓이면 Resource나 Area로 이동한다

## IPARA란?

**I**nbox + **P**rojects + **A**reas + **R**esources + **A**rchives

원래 PARA는 Tiago Forte가 만든 정리 시스템인데, 앞에 Inbox가 추가된 버전이에요.

| 폴더 | 특징 | 예시 |
|------|------|------|
| Inbox | 처리 전 | 방금 떠오른 생각 |
| Project | 기한 O, 목표 O | GameOver 알파 완성 |
| Area | 기한 X, 지속 관리 | 게임 개발 실력, 독서 |
| Resource | 외부 자료 수집 | NDC 영상, 기술 문서 |
| Archive | 완료/비활성 | 끝난 프로젝트 |

## 워크플로우 — 실제로 어떻게 쓰는가

### 💨 빠른 캡처 (언제든지)
뭔가 떠오르면 → **Inbox에 던져두기**. 완벽하게 쓸 필요 없어요.

### 📅 일일 루틴 (매일)
Daily 노트 열기 → 할 일 작성 → 오늘 진행한 프로젝트 업데이트

### 🔄 주간 처리 (주 1회, 권장 일요일)
1. Inbox 열기
2. 각 메모를 읽고 알맞은 폴더로 이동 or 삭제
3. 프로젝트 Hub 노트 열어서 상태 업데이트
4. 이번 주 회고 한 줄 작성

### 📦 월간 정리 (월 1회)
완료된 프로젝트나 더 이상 안 보는 노트를 Archive로 이동

## 템플릿 사용 방법

옵시디언에서 `Cmd+P` (Mac) 또는 `Ctrl+P` (Windows) → "Templates: Insert template" 선택

| 상황 | 사용 템플릿 |
|------|-----------|
| 오늘 일기/할 일 | `Daily Note` |
| 게임 개발 기록 | `Game Dev Log` |
| 책 읽고 정리 | `Book Review` |
| 아이디어 떠올랐을 때 | `Idea Note` |
| 팀 회의 후 | `Meeting Notes` |
| 짧은 생각 메모 | `ZettelKasten` |

## 자주 하는 실수

**❌ Inbox를 안 비움**  
→ 주 1회 30분만 투자하면 충분해요. Inbox에 20개 넘어가면 경고 신호예요.

**❌ 너무 완벽하게 쓰려 함**  
→ 메모는 초안이에요. 완성도보다 기록이 중요해요.

**❌ 폴더를 너무 세세하게 나눔**  
→ 폴더는 지금 구조로 충분해요. 폴더 만드는 데 시간 쓰지 마세요.

**❌ Project와 Area를 헷갈림**  
→ "이게 끝나는 시점이 있나?" 라는 질문으로 구분하세요.  
   있으면 → Project, 없으면 → Area
