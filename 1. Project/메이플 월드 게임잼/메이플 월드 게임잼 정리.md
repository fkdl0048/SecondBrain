### 접근 방식

이 질문까지 오면 《아주 작은 습관의 힘》의 핵심을 거의 이해한 겁니다.

프로젝트는 **"목표"와 "시스템" 사이에 한 단계가 더 있습니다.**

> **목표 → 프로젝트 → 시스템 → 습관**

예를 들어,

> **10월까지 프로젝트 완성**

이건 너무 큽니다.

그래서 이렇게 쪼갭니다.

```
10월 프로젝트 완료
        │
        ▼
프로젝트 단계
        │
        ├─ 조사
        ├─ 설계
        ├─ 개발
        ├─ 테스트
        └─ 배포
        │
        ▼
주간 목표
        │
        ├─ 이번 주 API 완성
        ├─ UI 3개 제작
        └─ 테스트 작성
        │
        ▼
일일 시스템
        │
        ├─ 오후 8시 프로젝트 시작
        ├─ 90분 집중
        ├─ 오늘 할 Task 1~2개 완료
        └─ 작업 로그 작성
```

보면 **프로젝트도 결국 일일 시스템으로 귀결됩니다.**

---

## 예를 들어 앱 개발이라면

10월 마감

오늘 해야 하는 것은

- 로그인 화면 만들기
    

내일

- 회원가입 API 연결
    

모레

- 오류 수정
    

이렇게 하루에 처리 가능한 단위(Task)만 남겨두는 것입니다.

---

## 중요한 점

많은 사람이

> "프로젝트를 해야 한다."

라고 생각합니다.

하지만 시스템 관점에서는

> "오늘 프로젝트를 위해 90분 동안 딱 한 가지 Task를 한다."

가 맞습니다.

---

## 그래서 프로젝트에는 3개의 층이 있습니다.

### ① 프로젝트 관리

```
10월 출시
```

여기는 일정 관리입니다.

---

### ② Task 관리

```
□ 로그인
□ 회원가입
□ API 연결
□ 테스트
```

오늘 무엇을 할지 관리합니다.

---

### ③ 시스템

```
퇴근
↓
샤워
↓
커피
↓
옵시디언 실행
↓
Git Pull
↓
90분 집중
↓
작업 로그 작성
↓
Git Push
```

이게 자동으로 실행되는 것이 시스템입니다.

---

## 오히려 프로젝트일수록 습관이 더 중요합니다.

예를 들어 매일 90분씩 작업하면

```
90분 × 7일 = 10.5시간

10.5시간 × 12주

= 약 126시간
```

126시간은 웬만한 프로젝트 하나를 끝낼 수 있는 수준의 시간입니다.

반면,

> "이번 주말에 몰아서 해야지."

는 보통 피로, 일정 변화, 의욕 저하 때문에 무너지기 쉽습니다.

---

### 네 상황이라면 이렇게 설계할 것 같아요.

네가 지금 만들고 있는 삶의 시스템을 보면 이미 큰 그림이 보입니다.

- **매일:** 운동, 독서, 프로젝트 90분, 블로그 20분
    
- **매주:** 프로젝트 진행 상황 점검, 블로그 2편 발행
    
- **매월:** 프로젝트 마일스톤 달성
    
- **10월:** 프로젝트 완료
    

이렇게 **장기 목표를 직접 관리하지 않고, '오늘 실행할 시스템'만 관리**하는 것이 핵심입니다.

사실 이 원리는 프로젝트뿐 아니라 공부, 운동, 투자, 창업에도 그대로 적용됩니다. 결국 성과는 목표를 자주 떠올리는 사람이 아니라, **좋은 시스템을 반복하는 사람**에게 쌓이는 경우가 많습니다.

## 4. 목표 완수를 위한 작은 습관

**매일**

- 개발 시작 전 5분: 어제 기록한 "내일 할 일 3가지" 확인 후 바로 착수 (워밍업 시간 낭비 방지)
- 3시간 집중 후 최소 10분 휴식 — 몰아서 밤샘하지 않기 (완주가 목표이므로 번아웃 관리가 곧 리스크 관리)
- 세션 종료 전 3줄 일지: 오늘 한 일 / 막힌 부분 / 내일 할 일 3가지

**매주 (일요일 권장)**

- 로드맵 대비 진행률 점검, 늦어진 항목 있으면 범위 축소 여부 결정
- 심사 기준 3가지(IP 해석 40 / 완성도 40 / 지속가능성 20) 기준으로 현재 빌드 자가 채점
- 가능하면 주 1회 이상 다른 사람에게 플레이테스트 받기 (혼자 개발 시 시야가 좁아지는 것 방지)

**환경 세팅**

- 버그/아이디어를 바로 적을 수 있는 간단한 트래커 준비 (노션, 트렐로, 메모 앱 무엇이든 좋음)
- 작업 파일 정기 백업 (에디터 자체 버전 관리 기능 확인 후, 안 되면 수동 백업 주기 정하기)
